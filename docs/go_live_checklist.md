# SQL Agent — Go-Live Checklist

> 上線前依序完成所有項目。每項需有責任人簽核後才可繼續。

---

## 一、功能驗收

### 設計模式
- [ ] 建立新 session（無 DB）→ 可進入對話頁
- [ ] 建立新 session（含 DB 連線字串）→ 顯示「匯入 N 個資料表」
- [ ] 對話 3 輪以上 → AI 追問後回傳結構化 Schema
- [ ] 確認頁正確顯示需求摘要 + Schema 表格
- [ ] 點「返回補充需求」→ 回到對話頁，對話記錄不遺失
- [ ] 確認後四份文件並行產出，進度卡各自更新
- [ ] 文件頁：規格書 / ER 圖（Mermaid 渲染正確）/ DDL（語法高亮）/ 安全規劃
- [ ] DDL 一鍵複製功能正常
- [ ] 下載 ZIP：內含 4 個檔案，命名正確
- [ ] 版本歷史：多輪對話後確認頁顯示多個版本；可還原至舊版本

### 審查模式
- [ ] 建立審查 session → 60 秒內產出報告
- [ ] 報告含四段分析 + 整體評分（X/10）
- [ ] 審查頁可下載報告（.md）

### Session 管理
- [ ] 首頁列出所有 sessions，依建立時間倒序
- [ ] 狀態篩選（全部/進行中/待確認/已完成）正常
- [ ] 點擊 session 卡片正確跳轉至對應頁面

### Schema Diff
- [ ] 有現有 DB 的設計 session，確認頁顯示 Diff 區塊
- [ ] 新增/刪除/變更欄位分類正確

---

## 二、效能驗收

- [ ] API 回應時間（非 AI 呼叫）P95 < 200ms（以 k6 或 wrk 測試）
- [ ] 四份文件並行產出時間 P95 < 60s
- [ ] 10 個並發 session 同時對話無明顯降速
- [ ] 首頁載入時間（含 sessions 列表）< 1s

---

## 三、錯誤處理驗收

- [ ] 填入錯誤 DB 連線字串 → 顯示 inline 錯誤，不跳轉
- [ ] AI API 斷線 → 對話頁顯示錯誤訊息，不崩潰
- [ ] 文件產出某一份失敗 → 其他三份繼續；失敗進度卡顯示錯誤原因
- [ ] 網路中斷後 3 次重試失敗 → 顯示「請重新整理」提示
- [ ] 快速雙擊確認按鈕 → 第二次請求被拒絕（HTTP 400）

---

## 四、安全性檢查（v0.5 範圍）

- [ ] `.env` 不在版控中（`git ls-files | grep .env` 回傳空）
- [ ] DB 連線字串不出現在任何 API 回應中
- [ ] DB 連線字串不出現在應用程式 log 中
- [ ] LLM API token 不出現在前端 HTML 中
- [ ] 所有 HTML 輸出有 escape（避免 XSS）
