# SQL Agent — 專案概述文件（Project Charter）

## 一、專案背景

資料庫規格書是軟體開發不可或缺的技術文件，但在現行工作流程中，系統分析師與開發工程師通常需要花費大量時間手工撰寫 DDL、ER Diagram 與欄位說明。這些文件格式繁瑣、容易遺漏，且難以保持與實際設計同步。

SQL Agent 利用 AI 對話能力，讓使用者用自然語言描述資料表需求，自動完成規格書產出，大幅縮短文件作業時間。

---

## 二、專案目標

| 目標 | 可量測指標 |
|---|---|
| 縮短資料庫文件作業時間 | 從描述到取得完整文件 ≤ 5 分鐘 |
| 提升文件完整性 | 自動產出 DDL + ER Diagram + 規格書 + 安全規劃，無遺漏 |
| 降低技術門檻 | PM / SA 不需懂 SQL 即可使用 |
| 標準化輸出格式 | 所有文件符合統一格式，可直接移交開發 |

---

## 三、目標使用者

| 角色 | 職責 | 使用 SQL Agent 的情境 |
|---|---|---|
| 系統分析師（SA） | 整理系統需求並產出技術規格 | 新專案啟動時設計多個資料表 |
| 專案經理（PM） | 協調需求並確認設計可行性 | 確認 AI 整理的設計摘要是否符合需求 |
| 後端開發者（Dev） | 實作資料庫與 API 層 | 快速建立新功能的資料表草稿 |
| 系統管理員（Admin） | 管理平台帳號與設定 | 管理使用者帳號與查閱系統日誌 |

---

## 四、專案範圍

### 包含（In Scope）

- 網頁平台：對話式需求收集、確認頁、文件查閱頁
- AI 驅動的資料庫規格書產出（DDL / ER Diagram / 規格書 / 安全規劃）
- 現有 PostgreSQL 資料庫結構匯入與審查模式
- Schema 版本管理（多版本回溯）
- Schema Diff（設計版本 vs 現有 DB 差異比對）
- 文件下載（單檔 .md/.sql / 全部 .zip）

### 不包含（Out of Scope）

- 直接連線目標資料庫並執行 DDL
- 多人協作同一 session
- 手機版 RWD（以桌面版 1280px+ 為主）
- 分享唯讀連結
- Word / PDF 匯出格式
- 多語言介面（僅繁體中文）

## 五、成功標準

- [ ] 從開始對話到取得全部四份文件，正常情況 ≤ 5 分鐘
- [ ] 文件格式符合規格：DDL 可直接在 PostgreSQL 執行，Mermaid 圖可正確渲染
- [ ] 支援 100 個並發 session 而不影響回應時間（< 3 秒 API 回應）
- [ ] 所有 API 端點有完整的 OpenAPI 規格文件
- [ ] 系統可在無原始開發者介入的情況下部署與維護
