# SQL Agent — 需求規格書

## 一、功能性需求

### FR-01：新建對話 Session

- 使用者可建立新的「設計模式」session（無需 DB 連線）
- 使用者可建立「審查模式」session（需提供 PostgreSQL 連線字串）
- Session 標題可自訂，預設為「未命名設計」
- 建立時可選擇匯入現有 PostgreSQL DB 結構作為設計參考

### FR-02：需求收集對話

- 使用者輸入自然語言需求，AI 進行多輪追問
- AI 在每輪回覆中追問欄位型態、主鍵、外鍵、索引等細節
- AI 判斷需求完整後，自動解析出結構化 Schema 並附加需求摘要
- 支援隨時返回對話修改需求（從確認頁返回）
- 右側進度面板即時顯示已收集的資料表與欄位

### FR-03：Schema 確認

- 確認頁左欄顯示 AI 整合雙方對話的需求摘要（3–6 條要點）
- 確認頁右欄以表格顯示 AI 理解的結構化 Schema
- 若 session 有匯入現有 DB，自動計算設計 Schema vs 現有 DB 的差異（新增/刪除/變更欄位）
- 支援多版本管理：每次 Schema 更新自動快照，最多保留 10 版，可還原任意版本

### FR-04：文件產出

- 確認後系統並行產出四份文件：
  - `01_specification.md`：規格書與資料字典
  - `02_er_diagram.md`：Mermaid ER Diagram
  - `03_ddl.sql`：PostgreSQL DDL + 索引 + Migration + Seed
  - `04_security_plan.md`：索引策略、存取控制、敏感欄位建議
- 產出進度即時顯示（每份文件獨立狀態：等待/產出中/完成/失敗）
- 全部完成後可在頁面預覽並下載

### FR-05：文件查閱

- 文件頁支援頁籤切換（規格書 / ER 圖 / DDL / 安全規劃）
- ER Diagram 頁籤直接渲染 Mermaid 圖形
- DDL 頁籤含語法高亮與一鍵複製
- 支援下載單份文件或全部打包為 `.zip`

### FR-06：審查模式

- 匯入現有 PostgreSQL DB 後，AI 自動分析並產出審查報告
- 審查報告含四段：設計一致性、資料完整性、效能考量、安全性
- 每段 3–5 條具體建議，格式：`資料表（欄位）：問題 → 建議`
- 報告含整體評分（X/10）與總評
- 支援下載報告（.md）

### FR-07：Session 管理

- 首頁列出所有 session，依建立時間倒序排列
- 支援狀態篩選：全部 / 進行中 / 待確認 / 已完成
- 不同模式（設計/審查）以不同圖示區分

## 二、非功能性需求

### NFR-01：效能

| 指標 | 目標 |
|---|---|
| API 回應時間（非 AI 呼叫） | P95 < 200ms |
| AI 訊息回覆時間 | P95 < 30s（受 LLM API 限制） |
| 文件產出時間 | P95 < 60s（四份並行） |
| 並發 session 數 | ≥ 100 |

### NFR-02：可靠性

- 文件產出失敗時，前端顯示具體錯誤訊息
- 網路中斷後，前端 3 次重試失敗顯示「請重新整理」
- Worker 執行中斷不影響其他 session

### NFR-03：安全性（待實作）

- 所有 API 需驗證使用者身份（JWT）
- 使用者只能存取自己的 session
- DB 連線字串不得以明文記錄於 log
- 敏感輸出（password、token 等欄位）不得洩漏至前端顯示

### NFR-04：可維護性

- 所有 API 端點有 OpenAPI 3.0 規格文件
- 所有關鍵操作寫入結構化 log（JSON 格式）
- 環境設定全部透過 `.env` 管理，不得 hardcode

### NFR-05：相容性

- 瀏覽器支援：Chrome 120+、Firefox 120+、Edge 120+
- 螢幕解析度：桌面版 1280px+ 為主
- 後端：Python 3.11+、PostgreSQL 14+（目標 DB）

## 三、資料需求

| 資料項目 | 來源 | 更新頻率 | 保留期限 |
|---|---|---|---|
| Session 對話記錄 | 使用者輸入 | 即時 | 永久（軟刪除） |
| 設計 Schema 版本 | AI 解析 | 每次確認 | 最多 10 版 |
| 文件產出內容 | AI 生成 | 每次產出 | 永久 |
| 現有 DB 結構快照 | PostgreSQL | 每次匯入 | 隨 session |
| 系統操作日誌 | 系統自動 | 即時 | 90 天 |

## 四、整合需求

| 系統 | 類型 | 說明 |
|---|---|---|
| Pensieve AI API | AI LLM | 所有 AI 對話與文件生成的核心服務 |
| PostgreSQL | 資料庫 | 使用者可選擇匯入的現有資料庫結構 |
| 瀏覽器 Clipboard API | Web API | DDL 一鍵複製功能 |
| Mermaid.js | 前端函式庫 | ER Diagram 即時渲染 |
